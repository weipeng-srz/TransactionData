# 数据格式与计算口径

Web 页面通过同域 API 获取数据。行情和新闻路由返回 UTF-8 CSV，财务与实时行情路由返回 JSON。

## 历史行情 CSV

`POST /api/local-stock-data` 返回带 BOM 的 UTF-8 CSV。

### 文件结构

1. 第一行 `#META`：股票代码、名称、价格口径、数据级别、时间精度、成交金额口径、数据来源和采集时间。
2. 每个交易日一行 `#DAY`：交易日期和当日前复权因子。
3. 一行表头。
4. 每个交易日由开、高、低、收四个价格点合成的日 K 代理记录。

### 主要字段

| 字段 | 说明 |
| --- | --- |
| `交易日期` | `YYYY-MM-DD` |
| `成交时间` | 用于映射 OHLC 的合成时间点，不代表真实逐笔发生时间 |
| `数据序号` | 文件内单日顺序 |
| `股票代码`、`股票名称` | 标的身份 |
| `原始成交价格(元)` | 上游日 K 返回的未复权价格 |
| `前复权成交价格(元)` | `原始成交价格 ÷ 当日前复权因子` |
| `成交量(股)` | 日成交量在四个合成点之间分配后的代理值 |
| `成交金额估算(元)` | `原始成交价格 × 代理成交量` |
| `数据级别` | 固定标记为 HTTPS 日 K 聚合行情 |

这些记录用于构建 K 线和研究指标，不是逐笔成交，也不应被解释为真实盘中资金流。

## 新闻 CSV

`POST /api/local-stock-news` 返回以下字段：

| 字段组 | 字段 |
| --- | --- |
| 标的 | 股票代码、股票名称 |
| 来源 | 检索入口、频道、媒体来源、原文链接 |
| 时间 | 发布时间、采集时间 |
| 相关性 | 相关性得分 |
| 情绪 | 情绪倾向、情绪得分、正向词、负向词 |
| 内容 | 新闻标题、新闻摘要 |

来自网页的文本会防护常见电子表格公式注入前缀。情绪结果来自本地关键词规则，只适合初筛，不能准确理解反讽、否定、语境和事件长期影响。

## JSON 接口

- `/api/local-stock-lookup`：规范化后的股票代码和名称。
- `/api/local-stock-financials`：报告期指标、估值快照、分红和三张报表分析。
- `/api/realtime-market`：最新价格、涨跌幅、分钟 K 线和五档盘口。
- `/api/research-state`：用户研究状态。

## 缓存与兼容性

历史行情和新闻响应成功后会写入浏览器 Cache API。网络失败时，页面最多使用七天内的只读缓存。服务端返回格式发生变化时，应保持已有字段语义，并同时更新解析器和回归测试。
