# 数据格式与计算口径

## 历史分笔 CSV

`stock-ticks` 默认输出 `data.csv`，编码为 UTF-8。

### 文件结构

1. 第一行 `#META`：文件级元数据，包括股票代码、名称、最新流通 A 股本及生效日，以及价格、时间和序号口径。
2. 每个交易日一行 `#DAY`：交易日期、当日前复权因子、流通 A 股本及其生效日。
3. 一行表头。
4. 按交易日和源数据顺序排列的成交明细。

### 成交字段

| 字段 | 说明 |
| --- | --- |
| `交易日期` | `YYYY-MM-DD` |
| `成交时间` | 源数据时间，历史分笔通常只有分钟精度 |
| `数据序号` | 文件内单日顺序，不是交易所成交序号 |
| `原始成交价格(元)` | 数据源返回的未复权价格 |
| `前复权成交价格(元)` | `原始成交价格 ÷ 当日前复权因子` |
| `成交量(股)` | 成交股数 |
| `成交金额(元)` | `原始成交价格 × 成交量` |
| `性质` | 买盘、卖盘、中性盘或其他 |
| `原始性质代码` | 数据源原始标记，用于复核 |
| `交易时段` | 正常交易或盘后交易等分类 |

前复权价格用于构建连续价格序列；资金流和大额成交分析应使用原始成交金额。换手率应使用对应 `#DAY` 中当日适用的流通 A 股本。

### 时间边界

程序按 Asia/Shanghai 时区判断完整交易日。北京时间 16:00 之前从前一个自然日开始回溯；16:00 之后可以把当天作为候选完整交易日。没有成交记录的周末、休市日和无效候选会跳过。

## 新闻 CSV

`stock-news` 默认输出 `news.csv`：

| 字段组 | 字段 |
| --- | --- |
| 标的 | 股票代码、股票名称 |
| 来源 | 检索入口、频道、媒体来源、原文链接 |
| 时间 | 发布时间、采集时间 |
| 相关性 | 相关性得分 |
| 情绪 | 情绪倾向、情绪得分、正向词、负向词 |
| 内容 | 新闻标题、新闻摘要 |

来自网页的文本会防护常见电子表格公式注入前缀。情绪结果来自本地关键词规则，只适合初筛，不能准确理解反讽、否定、语境和事件长期影响。

## 兼容性

Web 解析器仍能读取缺少新元数据的旧版 CSV，但会明确把旧数据中的资金流标记为前复权金额代理。新增字段时应保持已有字段语义不变，并在测试中覆盖新旧两种格式。

## 输出安全

- 目标目录会自动创建。
- 数据完整写入并刷新后才替换目标文件。
- CSV、压缩包和 `result/` 默认被 Git 忽略，避免把抓取数据误提交到公开仓库。
